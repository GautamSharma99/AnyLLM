# Implement Cross-CLI Integration System for anyllm

## Goal

Refactor `anyllm` so that:

1. `anyllm init` becomes completely CLI-agnostic and only initializes project state.
2. Add a new `anyllm install` command that installs slash command integrations into supported AI coding CLIs.
3. Users should be able to switch between Claude Code, Codex, OpenCode, Agy, Kiro, etc. while keeping the same command muscle memory:

```text
/anyllm-init
/anyllm-pack
/anyllm-repack
/anyllm-prime
/anyllm-push
/anyllm-status
```

The `.anyllm` folder is the portable project memory. Integrations are just UX sugar.

---

# Desired User Experience

First time:

```bash
pip install anyllm
anyllm install --all
```

Output:

```text
✓ Claude Code detected
✓ Codex detected
✓ OpenCode detected

Installing integrations...
✓ Claude Code integration installed
✓ Codex integration installed
✓ OpenCode integration installed
```

Then in ANY supported CLI:

```text
/anyllm-pack
/anyllm-prime
/anyllm-push
```

all work identically.

# Required Refactor

## Current behavior

```text
anyllm init
├── create .anyllm
└── install ~/.claude/commands
```

## New behavior

```text
anyllm init
└── create .anyllm
```

and

```text
anyllm install
└── install CLI integrations
```

No CLI-specific code should remain inside `init`.

# Detection

Implement automatic detection:

```python
integrations = [
    ClaudeIntegration(),
    CodexIntegration(),
    OpenCodeIntegration(),
    AgyIntegration(),
    KiroIntegration(),
    KiloIntegration(),
]
```

Each integration should detect whether its CLI is installed by checking common config directories or binaries on PATH.

Examples:

```python
shutil.which("claude")
shutil.which("codex")
shutil.which("opencode")
```

and/or known config directories.

Detection should never raise.

# Slash Commands

All integrations should install wrappers for:

```text
/anyllm-init
/anyllm-pack
/anyllm-repack
/anyllm-prime
/anyllm-push
/anyllm-status
/anyllm-log
/anyllm-diff
```

These wrappers should ONLY execute the actual CLI:

```bash
anyllm init
anyllm pack
anyllm repack
anyllm prime
anyllm push
anyllm status
anyllm log
anyllm diff
```

No business logic should live inside the wrappers.

---

# Important Design Constraint

The slash command names must be identical across all supported coding agents.

A user should be able to:

```text
Claude Code
    ↓
/anyllm-pack

switch to Codex
    ↓
/anyllm-prime

switch to OpenCode
    ↓
/anyllm-push
```

without learning different commands.

---

# Backward Compatibility

Existing Claude Code users should not break.

If Claude integration already exists:

```text
~/.claude/commands
```

preserve current behavior by moving the installation logic into:

```python
ClaudeIntegration.install()
```
